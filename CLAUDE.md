# CLAUDE.md - 프로젝트 개발 명세 및 가이드라인

디지털 수제 공방 **code-craftworks**에 오신 것을 환영합니다. 본 문서는 자녀 증여세 10년 역산 시뮬레이터인 **child-gift-bench** 유틸리티를 개발하기 위한 절대적인 블루프린트입니다. 클로드(Claude)는 새로운 코드를 작성하거나 수정할 때마다 이 명세서를 참조하여 디자인 일관성과 세법 엔진의 정확성을 유지해야 합니다.

## 🎨 디자인 시스템 및 UI/UX 가이드라인

본 애플리케이션은 극도로 미니멀하고 세련된 단일 페이지 웹앱(SPA) 형태로 구축되어야 합니다. 불필요한 장식을 배제하고 장인정신이 묻어나는 깔끔하고 직관적인 레이아웃에 집중합니다.

* **배경색 (캔버스):** `오프 화이트(Off-White)` (예: `#F8F9FA` 또는 `#F5F5F3`) - 눈이 편안하고 따뜻하며 정갈한 느낌을 주는 베이스 배경.
* **주 강조색 (신뢰와 구조):** `코발트 블루(Cobalt Blue)` (예: `#0047AB` 또는 `#1E40AF`) - 차트의 기본 구조, 비과세 안전 구간 타임라인, 한도 지표 등에 사용.
* **부 강조색 (인터랙션 및 경고):** `플레임 오렌지(Flame Orange)` (예: `#E65C00` or `#EA580C`) - 슬라이더 조절 노브(Knob), 실시간 마우스 호버(Hover), 세금 발생 한도 초과 경고 등에 사용.
* **스타일 테마:** 초미니멀리즘. 넓은 여백(Whitespace)을 활용하고, 산세리프(Sans-serif) 계열의 얇고 선명한 타이포그래피를 채택하며, 마우스로 밀고 당길 수 있는 직관적인 타임라인 컨트롤러를 배치합니다.

## 🧮 핵심 증여세 엔진 사양 (Core Tax Engine)

계산 엔진은 대한민국의 상속세 및 증여세법 규정을 엄격히 준수하여 실시간 연산을 수행해야 합니다.

### 1. 10년 롤링 윈도우(Look-back Window) 알고리즘
* **슬라이딩 타임라인:** 사용자가 지정한 임의의 기준일($T_{current}$)이 설정되면, 엔진은 기준일로부터 과거 10년 범위 내에 존재하는 모든 기신고 증여가액을 누적 합산합니다.
    $$\lfloor T_{current} - 10\text{년}\rfloor \le T_{gift} \le T_{current}$$
* **한도 실시간 부활:** 특정 증여일($T_{gift}$)로부터 정확히 10년(Calendar 연도 기준)이 지나 타임라인 윈도우 왼쪽 밖으로 벗어나는 순간, 해당 금액만큼의 비과세 공제 한도는 즉시 부활(리셋)됩니다.

### 2. 수증자(자녀) 신분별 비과세 공제 한도 (10년 누적 기준)
* **미성년 자녀:** 20,000,000 원
* **성년 자녀:** 50,000,000 원
* **법적 성년 기준일 정의:** 대한민국 민법에 의거하여, 수증자는 정확히 **'만 19세가 되는 해의 생일' 당일**부터 성인으로 판정됩니다.
    * *예시:* 2007년 10월 1일생인 자녀는 2026년 9월 30일까지 만 18세이므로 미성년 한도(2,000만 원)가 적용되며, 만 19세 생일이 되는 **2026년 10월 1일 당일부터 성년 한도(5,000만 원)가 공식적으로 트리거**됩니다.

### 3. 직계존속 그룹 합산 규칙 (Group Consolidation)
* 증여세 공제 한도는 증여재산공제 규정에 따라 개별 인원이 아닌 **'직계존속 그룹 전체(부모, 조부모, 외조부모 등)'가 하나의 바구니(한도)**를 공유합니다.
    * *예시:* 동일한 10년 주기 내에 할아버지가 미성년 자녀에게 이미 1,000만 원을 증여했다면, 부모가 세금 없이 추가로 줄 수 있는 남은 비과세 잔여 한도는 자동으로 1,000만 원으로 제한(Throttling)됩니다.

### 4. 혼인·출산 통합 공제 특례 (최신 개정 세법)

#### 4-1. 혼인 공제 기한 (총 4년 바구니)
* **기준점 ($T_{marriage}$):** 법적 **혼인신고일** (결혼식 날짜 아님)
* **범위:** 혼인신고일 **전후 2년** 이내 증여 전부 해당
    $$T_{marriage} - 2\text{년} \le T_{gift} \le T_{marriage} + 2\text{년}$$

#### 4-2. 출산(입양) 공제 기한 (총 2년 바구니)
* **기준점 ($T_{birth}$):** 자녀 **출생일 또는 입양신고일** — 역산 없음
* **범위:** 출생(입양)일 이후 **2년 이내** 증여만 해당
    $$T_{birth} \le T_{gift} \le T_{birth} + 2\text{년}$$

#### 4-3. ⚠️ 핵심 제한: 혼인+출산 통합 1억 원 한도 배리어
* **혼인 공제와 출산 공제는 합산하여 평생 통합 1억 원이 최대 한도.** 각각 1억씩 총 2억이 아님.
* **추가 면제 보너스:** 기본 성년 공제 한도(5,000만 원) 외에 혼인+출산 합산 최대 **1억 원 추가 공제**
* **최대 비과세 한도:** 5,000만 + 1억 = **1억 5,000만 원**
* **계산 공식 (세금 엔진):**
    $$\text{특례 보너스} = \min(\text{혼인 증여액}_{\text{active}} + \text{출산 증여액}_{\text{active}},\ 10{,}000\text{만})$$
* **시나리오 A** (결혼 후 출산): 혼인 공제로 1억 수령 → 출산 공제 잔여 **0원**
* **시나리오 B** (혼전 출산): 출산 공제로 7,000만 수령 → 혼인 공제 잔여 **3,000만 원**

#### 4-4. 시뮬레이터 구현 사양
* 혼인/출산 이벤트는 각각 **날짜 + 금액**으로 입력 (`specialEvents.marriage = {date, amount}`)
* 특례 증여는 정규 10년 누적 합산(`asum`)에도 포함되어 타임라인에 별도 바로 표시
* 타임라인 색상: 혼인(💍 보라 `#7C3AED`) · 출산(👶 청록 `#0891B2`)
* 자석 스냅: 혼인신고일·출생(입양)일이 기준날짜 스냅 포인트에 추가됨

### 5. 사전증여재산 상속세 역합산 배리어 (상증세법 제13조)
* 자녀에게 미리 증여한 재산은 증여일로부터 **10년 이내에 증여자가 사망할 경우, 상속세 과세가액에 강제로 다시 합산**되어 상속세율로 정산됩니다.
* 시스템 인터페이스는 이 '10년의 유통기한'을 안전하게 버텨내어 상속세 레이더망을 탈출시키는 것이 성공적인 사전 증여 자산 최적화의 핵심 지표임을 시각적으로 강조해야 합니다.

## 🛠️ 기술 스택 및 구현 제약 사항 (Tech Stack Constraints)

클로드(Claude)는 코드를 수정하거나 기능을 추가할 때, 외부 라이브러리에 의존하지 않는 현재의 **'장인식 수제 아키텍처(Bespoke Pure Tech)'**를 반드시 유지해야 합니다.

1. **의존성 제로 (Zero-Dependency Vanilla Architecture)**
   - **No Frameworks:** React, Vue, Next.js 등을 절대 사용하지 않습니다.
   - **No External Libraries:** Tailwind CSS CDN, jQuery, D3.js 등을 일절 추가하지 않고, 오직 브라우저 내장 API와 표준 기술로만 구동합니다.
   - **단일 파일 제약:** 모든 HTML 마크업, CSS 스타일, 자바스크립트 로직, SVG 렌더러는 단 하나의 파일(`index.html`) 내에 완벽히 통합되어야 합니다.

2. **디자인 테마 및 스타일링 사양 (CSS)**
   - 첨부된 오프 화이트 공방 템플릿의 CSS 룰셋과 클래스 구조(`.card`, `.lbl`, `.field-label`, `.btn-sm` 등)를 계승합니다.
   - **배경색:** 전체 캔버스는 부드러운 오프 화이트 계열로 유지합니다.
   - **강조 포인트:** 주요 수치, 활성 타임라인 영역, 헤더 등에는 **코발트 블루(Cobalt Blue)**를 사용하고, 슬라이더 노브 및 중요 주의 경고 등에는 **플레임 오렌지(Flame Orange)**를 심미적으로 조화롭게 배치합니다.

3. **고성능 동적 렌더링 및 상태 관리 (State & Rendering)**
   - 데이터가 변경되거나 사용자가 타임라인을 드래그할 때, 가상 돔(Virtual DOM) 없이 전체 HTML 구조를 새롭게 조립하여 주입하는 `render()` 및 `derive()` 기반의 단방향 데이터 흐름을 유지합니다.
   - 모든 가변 상태(`gifts`, `simDate`, `birthDate`, `chartYS`, `chartYE`)는 `localStorage`와 실시간 동기화(영속성 저장)되어야 합니다.

4. **수제 SVG 시각화 타임라인 모듈**
   - 상단 10개년 증여 수명 타임라인은 Canvas나 차트 라이브러리 대신, `<svg>` 엘리먼트 내부에 직접 `<rect>`, `<line>`, `<circle>` 태그를 계산된 좌표(`toX`)로 찍어내는 순수 벡터 드로잉 방식을 고수합니다.
   - 타임라인 드래그 핸들이나 마우스 클릭 시, 주요 날짜(성년 전환일, 10년 만료일) 좌표에 근접하면 부드럽게 고정되는 **'자석 스냅(Magnet Snapping)'** 알고리즘이 완벽히 작동해야 합니다.
